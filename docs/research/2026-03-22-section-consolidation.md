# Dashboard 섹션 중복/유사성 분석 및 통폐합 연구

## 개요

Dashboard.tsx(2,516줄)의 31개 섹션에 대해 중복/유사 콘텐츠를 분석하고 통폐합 방안을 도출한 연구 문서.

- **분석 대상:** results/ 디렉토리 50개 JSON 파일 + Dashboard.tsx 31개 섹션 렌더링
- **분석 날짜:** 2026-03-22

---

## 1. 강한 중복 — 즉시 통폐합 대상

### 그룹 A: 신호 기반 (3개 → 1개)

| 섹션 ID | 섹션명 | 핵심 정보 |
|---------|--------|----------|
| `signals` | AI 주목 종목 | 매수/적극매수 신호 종목 배지 리스트 |
| `consecutive` | 연속 시그널 | 매수+대장주 동시(AND) / 하나 이상(OR) 조건 |
| `cross` | 교차 신호 | vision+KIS 이중 확인, 테마, 장중 변화율 |

**중복 근거:**
- 3개 모두 "이 종목을 매수할까?"에 대한 답변
- 같은 종목이 signals에도, consecutive에도, cross에도 반복 등장
- 사용자 입장에서 3개 섹션을 따로 보는 의미 희박

**통합 제안:** "AI 매매 신호" 1개 섹션으로 통합. 내부에서 신호 강도별 필터 (적극매수 > 매수 > 대장주 교차) 제공.

---

### 그룹 B: 시뮬레이션 (2개 → 1개)

| 섹션 ID | 섹션명 | 핵심 정보 |
|---------|--------|----------|
| `simulation` | 전략 시뮬레이션 | 현재 전략별 거래수, 승률, 평균수익률 |
| `sim_history` | 시뮬레이션 히스토리 | 과거 날짜별 동일 지표 (거래수, 승률, 수익률) |

**중복 근거:**
- 동일한 지표(거래수, 승률, 수익률)를 현재/과거로 나눠 2번 표시
- 가장 강한 통합 후보

**통합 제안:** "전략 시뮬레이션" 1개 섹션에 현재 성과 + 히스토리 추이를 함께 표시.

---

### 그룹 C: 스코어/랭킹 (2개 → 1개)

| 섹션 ID | 섹션명 | 핵심 정보 |
|---------|--------|----------|
| `smartmoney` | 스마트 머니 TOP | 외국인+기관 매수 패턴 종합 스코어 |
| `valuation` | 밸류에이션 스크리너 | PER/PBR/ROE 기반 밸류 스코어 |

**중복 근거:**
- 둘 다 "종목 TOP 랭킹" 형태의 리스트
- 스마트머니 = 수급 관점, 밸류에이션 = 펀더멘탈 관점이지만 사용자에게는 "좋은 종목 순위"로 인식
- 같은 종목이 양쪽에 동시 등장 가능

**통합 제안:** "종목 스코어보드" 1개 섹션에 수급 스코어 + 밸류 스코어를 탭/필터로 전환.

---

### 그룹 D: 수급 흐름 (4개 → 1개)

| 섹션 ID | 섹션명 | 핵심 정보 | 관점 |
|---------|--------|----------|------|
| `heatmap` | 시간대별 수익률 | 시간대별 외국인/기관 순매매(억원) | 시간축 |
| `intraday_flow` | 장중 종목별 수급 | 종목별 외국인/기관 순매수(주) | 종목축 |
| `sector` | 테마별 자금 흐름 | 테마별 외국인 순매수(천주) | 테마축 |
| `member` | 증권사 매매 동향 | 증권사별 외국인 순매수 + 매수 TOP5 | 증권사축 |

**중복 근거:**
- 4개 모두 "외국인/기관 자금이 어디로 흐르는가?"에 대한 답변
- 같은 데이터를 시간/종목/테마/증권사 관점에서 반복

**통합 제안:** "자금 흐름" 1개 섹션에 4개 관점을 탭으로 전환.

## 2. 중간 중복 — 통합 검토 대상

### 그룹 E: 테마 분석 (3개 → 1개)

| 섹션 ID | 섹션명 | 핵심 정보 |
|---------|--------|----------|
| `lifecycle` | 테마 라이프사이클 | 테마 단계(탄생/성장/과열), 평균수익률 |
| `propagation` | 테마 전이 예측 | 리더→팔로워 전파, 예상 시간 |
| (`briefing` 일부) | AI 브리핑 테마 섹션 | 주목 테마, 카탈리스트 |

**중복 근거:**
- 3곳에서 테마 분석을 반복
- lifecycle: 단계, propagation: 전파, briefing: AI 해석 → 관점은 다르지만 "테마"라는 주제가 동일

**통합 제안:** "테마 인사이트" 1개 섹션에 단계 + 전파 + AI 해석 통합. briefing의 테마 부분은 별도 유지 가능 (AI 브리핑 성격상).

---

### 그룹 F: 이상치 감지 (3개 → 1개)

| 섹션 ID | 섹션명 | 감지 기준 |
|---------|--------|----------|
| `anomaly` | 이상 거래 감지 | 거래량 N배, RSI, 등락률 10%+ |
| `divergence` | 거래량-가격 괴리 | 거래량↑가격↓ 또는 거래량↓가격↑ |
| `gap` | 갭 분석 | 전일 종가 대비 시가 갭 |

**중복 근거:**
- 3개 모두 "비정상적 거래 패턴" 감지
- anomaly: 절대적 이상치, divergence: 상대적 괴리, gap: 시가 갭

**통합 제안:** "이상 거래 감지" 1개 섹션에 이상 유형별(거래량/괴리/갭) 탭.

## 3. 약한 중복 — 현행 유지

### 독립성이 명확한 섹션들

| 섹션 ID | 이유 |
|---------|------|
| `premarket` | 장 시작 전 전용, 시간대 특화 |

| `insider` | 내부자 거래 공시, 고유 데이터 |
| `events` | 이벤트 일정, 고유 데이터 |
| `earnings` | 실적 공시 일정, 고유 데이터 |
| `orderbook` | 실시간 호가, 고유 관점 |
| `auction` | 동시호가 전용, 시간대 특화 |
| `consensus` | 애널리스트 목표가, 고유 데이터 |

| `consistency` | 신호 안정성 추적, 보조 지표 |
| `paper_trading` | 모의투자 성과, 고유 데이터 |
| `trading_value` | 거래대금 순위, 고유 관점 |
| `journal` | 매매 일지, 고유 데이터 |
| `mentor` | AI 조언, 고유 성격 |
| `forecast` | 예측 적중률, 성과 추적 |
| `squeeze` | 역발상 신호, 공매도 특화 |
