# 주식 투자 활용 프로젝트 아이디어 연구

> 기존 프로젝트 `theme_analysis`, `signal_analysis`의 소스코드, 인프라, 결과물을 기반으로 한 5개 신규 프로젝트 아이디어

## 기존 자산 요약

### theme_analysis (테마 분석기)
- **수집 데이터**: 거래량/거래대금/등락률 TOP30, 종목별 3일 이상 시세, PER/PBR/시가총액, 공매도 비율, 투자자별 매매동향, 분봉 데이터, 거래량 프로파일, 매크로 지표(환율, 지수 이동평균)
- **AI 분석**: Gemini 기반 테마 식별(테마명 + 대장주 + 뉴스 근거), 3개 시간축 테마 예측(당일/단기/장기)
- **출력물**: `latest.json`(24.7MB 일일 스냅샷), `theme-forecast.json`(테마 예측), 90일+ 히스토리, 모의투자 결과
- **인프라**: GitHub Actions 11개 워크플로우, React 프론트엔드, 텔레그램 알림

### signal_analysis (SignalPulse)
- **수집 데이터**: KOSPI/KOSDAQ 상위 100종목 스크린샷, KIS API TOP30 상세(이동평균, 외국인/기관 수급, 호가, PER/PBR), Fear & Greed Index, VIX
- **AI 분석**: Gemini 비전 분석(스크린샷 기반 기술적 분석), KIS 데이터 분석, 교차 검증(Combined), 5단계 매매신호(적극매수~적극매도)
- **출력물**: `vision_analysis.json`, `kis_analysis.json`, `combined_analysis.json`, 시뮬레이션 백테스트 결과, 30일 히스토리
- **인프라**: GitHub Actions 3개 워크플로우, React 프론트엔드, 텔레그램 알림

### 공유 인프라
KIS API, Gemini AI(5키 로테이션), Supabase, 텔레그램 봇, Naver News API, GitHub Actions, GitHub Pages

## 아이디어 1: 크로스 시그널 실시간 알림 시스템

### 개요
두 프로젝트의 분석 결과를 교차 검증하여 **고확신 매매 시점**을 실시간 텔레그램으로 알려주는 시스템.

### 핵심 가치
- **실시간성**: 장중 데이터 갱신 시마다 즉시 알림
- **분석 깊이**: 단일 시스템보다 두 시스템의 합의(consensus)가 더 신뢰도 높음
- **편의성**: 텔레그램 한 채널에서 핵심 정보만 수신

### 작동 방식
1. theme_analysis의 테마 대장주 + signal_analysis의 매매신호를 매칭
2. **일치 조건**: 테마 대장주이면서 적극매수/매수 신호 → "고확신 매수 후보"
3. **역방향 경고**: 보유 종목이 테마에서 이탈 + 매도 신호 → "이탈 경고"
4. 장중 5분봉/분봉 데이터 기반 실시간 모니터링

### 텔레그램 알림 예시
```
🔥 [고확신 매수 후보] 삼성SDI (006400)
━━━━━━━━━━━━━━━━━
테마: 2차전지 (테마 순위 #1, 대장주)
신호: 적극매수 (비전 85점, KIS 82점)
기준: MA정배열 + 외국인 3일 연속 순매수
현재가: 425,000원 (+3.2%)
━━━━━━━━━━━━━━━━━
```

### 활용할 기존 자산
| 자산 | 출처 |
|------|------|
| 테마 대장주 목록 | theme_analysis → `latest.json` (AI 테마 분석) |
| 매매 신호 | signal_analysis → `combined_analysis.json` |
| 분봉 데이터 | theme_analysis → `intraday-history.json` |
| 수급 데이터 | signal_analysis → KIS 외국인/기관 수급 |
| 텔레그램 봇 | 두 프로젝트 공유 인프라 |

### 기술 구현
- Python 스크립트 + GitHub Actions (장중 5~10분 주기)
- 두 프로젝트의 JSON 결과물을 읽어서 교차 매칭
- 텔레그램 봇 API로 알림 전송
- 알림 이력을 JSON으로 저장하여 중복 방지

### 난이도 & 기대효과
- **난이도**: ★★☆☆☆ (기존 결과물 조합, 신규 수집 불필요)
- **기대효과**: 두 독립 시스템의 합의 신호로 거짓 신호(false positive) 대폭 감소

## 아이디어 3: 테마 라이프사이클 트래커

### 개요
테마의 **탄생 → 성장 → 과열 → 쇠퇴** 생명주기를 추적하고, 현재 어떤 단계에 있는지 시각화하여 **진입/이탈 타이밍**을 제안하는 시스템.

### 핵심 가치
- **분석 깊이**: 테마를 단순 순위가 아닌 생명주기로 분석
- **편의성**: 한눈에 테마별 상태 파악 가능한 대시보드
- **실시간성**: 단계 전환 시 텔레그램 알림

### 작동 원리

theme_analysis의 90일+ 테마 히스토리 데이터를 활용하여 각 테마의 시계열을 분석:

| 단계 | 판단 기준 | 전략 제안 |
|------|----------|----------|
| **탄생** 🌱 | 신규 테마 출현, 거래량 급증 시작 | "초기 진입 기회, 대장주 중심 소량 매수" |
| **성장** 📈 | 연속 상위 랭크, 대장주 상승 지속, 관련주 확산 | "추세 추종 매수, 눌림목 매수 유효" |
| **과열** 🔥 | 거래량 최고치, 후발주까지 급등, 뉴스 과잉 | "신규 진입 자제, 보유분 분할 매도 시작" |
| **쇠퇴** 📉 | 대장주 하락 전환, 거래량 감소, 랭크 하락 | "잔여분 정리, 다음 테마 탐색" |

### 분석 데이터
```
테마별 시계열 데이터:
- 일별 테마 순위 변동 (theme_analysis history 90일)
- 대장주 주가/거래량 추이
- 테마 내 종목 수 변화 (확산도)
- 관련 뉴스 빈도/감성 변화
- signal_analysis 매매신호 분포 변화
```

### 대시보드 UI (웹)
- **버블 차트**: X축=테마 수명(일), Y축=현재 모멘텀, 크기=거래대금
- **테마 타임라인**: 각 테마의 단계 전환 이력
- **알림 설정**: 특정 테마의 단계 전환 시 텔레그램 알림

### 활용할 기존 자산
| 자산 | 출처 |
|------|------|
| 테마 히스토리 (90일) | theme_analysis → `frontend/public/data/history/` |
| 테마 예측 히스토리 (20일) | theme_analysis → `forecast-history/` |
| 대장주 시세 | theme_analysis → 종목별 시세 데이터 |
| 종목별 매매신호 추이 | signal_analysis → `history/` (30일) |
| 뉴스 감성 분석 | signal_analysis → 뉴스 sentiment 데이터 |

### 기술 구현
- Python: 테마 히스토리 파싱 + 단계 판정 로직
- Gemini: 단계 전환 판단의 보조(경계 사례 분석)
- React 페이지 추가 또는 독립 웹앱
- GitHub Actions: 일 1회 분석 후 결과 저장

### 난이도 & 기대효과
- **난이도**: ★★★☆☆ (히스토리 데이터 파싱 + 단계 판정 로직 설계)
- **기대효과**: 테마 투자의 가장 큰 리스크인 "과열 구간 진입"을 방지, 체계적 테마 로테이션

## 아이디어 4: 보유종목 실시간 리스크 모니터

### 개요
사용자가 등록한 **보유 종목 포트폴리오**를 실시간으로 모니터링하여, 위험 신호 발생 시 즉시 텔레그램으로 경고하는 시스템. "내가 가진 종목"에 초점을 맞춘 방어적 도구.

### 핵심 가치
- **실시간성**: 장중 위험 신호 즉시 알림
- **분석 깊이**: 개별 종목이 아닌 포트폴리오 전체의 리스크 분석
- **편의성**: 보유 종목만 추적하므로 노이즈 최소화

### 모니터링 항목

| 경고 유형 | 데이터 소스 | 알림 기준 |
|-----------|-----------|----------|
| **신호 악화** | signal_analysis | 매수 → 중립/매도 전환 |
| **테마 이탈** | theme_analysis | 소속 테마 순위 급락 또는 테마에서 제외 |
| **수급 반전** | KIS API 투자자 데이터 | 외국인/기관 3일 연속 순매도 전환 |
| **공매도 급증** | theme_analysis stock_criteria | 공매도 비율 5% 돌파 |
| **이동평균 이탈** | KIS API / 분봉 데이터 | 20일선 하향 이탈 |
| **거래량 이상** | 분봉 데이터 | 평균 대비 300% 이상 급증 (투매 징후) |
| **포트폴리오 상관관계** | 종합 분석 | 보유 종목 다수가 동일 테마/섹터 편중 경고 |

### 텔레그램 알림 예시
```
⚠️ [리스크 경고] 포트폴리오 알림
━━━━━━━━━━━━━━━━━

🔴 에코프로비엠 (247540) - 위험도 높음
• 신호 변동: 매수 → 매도 (비전 점수 85→45)
• 외국인 4일 연속 순매도 (-320억)
• 20일 이동평균선 하향 이탈
• 조치 제안: 분할 매도 검토

🟡 삼성SDI (006400) - 주의
• 소속 테마(2차전지) 순위 하락 (#1→#5)
• 공매도 비율 4.8% (경고 근접)
• 조치 제안: 추이 관찰, 손절선 설정

📊 포트폴리오 전체
• 2차전지 비중 65% → 섹터 편중 주의
• 오늘 전체 수익률: -1.2%
━━━━━━━━━━━━━━━━━
```

### 보유종목 등록 방식
- **방법 1**: 텔레그램 봇 대화형 (`/add 삼성SDI 10주 425000`)
- **방법 2**: JSON 설정 파일 직접 편집
- **방법 3**: KIS API 실계좌 연동 (자동 동기화)

### 활용할 기존 자산
| 자산 | 출처 |
|------|------|
| 매매 신호 변동 | signal_analysis → analysis JSON + history |
| 테마 소속 및 변동 | theme_analysis → AI 테마 분석 결과 |
| 수급 데이터 | theme_analysis → 투자자 매매동향 |
| 공매도/기준 데이터 | theme_analysis → stock_criteria 9요소 |
| 분봉/거래량 | theme_analysis → intraday-history |
| KIS 계좌 연동 | 두 프로젝트 → KIS API 인프라 |

### 기술 구현

### 난이도 & 기대효과

## 아이디어 5: 통합 모바일 투자 대시보드 (PWA)

### 개요
theme_analysis와 signal_analysis의 핵심 정보를 **모바일 최적화된 단일 화면**으로 통합하는 PWA(Progressive Web App). 현재 두 프로젝트가 각각 별도의 React 프론트엔드를 가지고 있어 정보가 분산되어 있는 문제를 해결.

### 핵심 가치
- **편의성**: 두 앱을 오가지 않고 한 화면에서 모든 정보 확인
- **분석 깊이**: 두 시스템의 데이터를 결합한 종합 뷰 제공
- **실시간성**: PWA 푸시 알림 + 자동 새로고침

### 화면 구성

#### 메인 대시보드 (한 화면)
```
┌─────────────────────────────┐
│ 📊 시장 상태 바                │
│ KOSPI 2,650(+1.2%) │ F&G 62 │
│ KOSDAQ 880(+0.8%)  │ VIX 18 │
├─────────────────────────────┤
│ 🔥 오늘의 TOP 테마             │
│ 1. 2차전지  2. AI반도체       │
│ (탭하면 소속 종목 + 신호 확인)   │
├─────────────────────────────┤
│ 🎯 고확신 종목 (크로스 시그널)   │
│ ┌──────┬──────┬──────┐      │
│ │삼성SDI│SK하이│LG에너│      │
│ │적극매수│ 매수 │ 중립 │      │
│ └──────┴──────┴──────┘      │
├─────────────────────────────┤
│ 📋 내 포트폴리오 요약           │
│ 오늘 수익: +45,000원 (+0.8%)  │
│ 경고: 에코프로 리스크 🔴       │
├─────────────────────────────┤
│ 📈 시뮬레이션 성과              │
│ 30일 누적: +12.5%            │
│ 오늘 적극매수 평균: +2.3%      │
└─────────────────────────────┘
```

#### 하위 페이지
- **테마 상세**: 테마별 종목 목록 + 각 종목의 signal_analysis 신호
- **종목 상세**: 테마 소속 + 매매신호 + 차트 + 뉴스 + 수급 통합 뷰
- **히스토리**: 일별 브리핑 아카이브 + 적중률 추이

### 활용할 기존 자산
| 자산 | 출처 |
|------|------|
| 전체 테마/종목 데이터 | theme_analysis → `latest.json` |
| 매매 신호 전체 | signal_analysis → 3개 analysis JSON |
| 매크로 지표 | 두 프로젝트 → macro/fear_greed/vix JSON |
| 시뮬레이션 결과 | 두 프로젝트 → simulation 데이터 |
| React 컴포넌트 | 두 프로젝트의 기존 React 코드 재활용 가능 |
| 배포 인프라 | GitHub Pages + GitHub Actions |

### 기술 구현
- React + TypeScript + TailwindCSS (기존 스택 동일)
- PWA: Service Worker + manifest.json (오프라인 + 푸시 알림)
- 두 프로젝트의 JSON을 fetch하여 통합 (CORS 처리 또는 빌드 시 복사)
- GitHub Actions: 두 프로젝트 분석 완료 후 자동 빌드/배포

### 난이도 & 기대효과
- **난이도**: ★★★★☆ (프론트엔드 개발 + 두 데이터 소스 통합 + PWA 설정)
- **기대효과**: 투자 의사결정 시간 단축, 정보 분산 문제 해결, 모바일에서 즉시 확인

## 추천 우선순위

### 1순위: AI 모닝 브리프 & 이브닝 리뷰
- **이유**: 기존 데이터를 그대로 활용, 신규 수집 불필요. Gemini 프롬프트 엔지니어링만으로 구현 가능. 매일 체감할 수 있는 가치를 즉시 제공. 적중률 추적으로 시스템 전체의 신뢰도도 검증 가능.

### 2순위: 크로스 시그널 실시간 알림
- **이유**: 가장 낮은 개발 난이도로 가장 빠르게 구현 가능. 두 프로젝트의 JSON 결과물을 읽어서 매칭하는 단순한 로직. 모닝 브리프와 함께 구현하면 시너지가 큼.

### 3순위 이후
- 보유종목 리스크 모니터 → 실용적이지만 포트폴리오 관리 기능이 추가로 필요
- 테마 라이프사이클 → 독창적이지만 히스토리 데이터 분석 로직 설계에 시간 소요
- 통합 PWA → 최종 목표로 적합하나 프론트엔드 개발량이 많음

### 추천 구현 순서
```
1단계: 모닝 브리프 + 크로스 시그널 알림 (동시 구현 가능, 1~2일)
2단계: 보유종목 리스크 모니터 (텔레그램 봇 확장)
3단계: 테마 라이프사이클 트래커 (분석 엔진)
4단계: 통합 PWA (1~3단계 결과물을 모두 포함)
```
